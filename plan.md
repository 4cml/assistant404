## 1. نظرة عامة على المشروع

مشروع **Assistant404**: تطبيق ويب (React + Vite) لعرض مواد الخطة الدراسية ومحاضراتها (ملفات PDF)، مع دعم:
- تصفح المواد (الكل / المفضلة / الأخيرة / الأكثر زيارة)
- بحث فوري (Fuse.js)
- عرض الخطة الدراسية كاملة مع الربط بصفحات المواد
- صفحة مادة تعرض المحاضرات مقسّمة (نظري / عملي / ملفات إضافية / أسئلة)

**البنية التقنية:**
- `src/App.jsx` — الراوتنغ الرئيسي
- `src/components/Sidebar.jsx` — القائمة الجانبية
- `src/pages/SubjectList.jsx`, `Subject.jsx`, `StudyPlan.jsx`
- `public/data/study-plan.json` — الخطة الدراسية الكاملة
- `public/data/subjects-index.json` — فهرس المواد المفعّلة
- `public/pdf/<subjectId>/subject.json` + `lectures.json` — بيانات كل مادة ومحاضراتها

## 4. المطلوب إنجازه — المراحل القادمة بالتفصيل

### المرحلة أ: استكمال تحويل Tailwind (أولوية عالية)
الهدف: التخلص من كل inline styles في المشروع لصالح Tailwind، تدريجيًا وملف بملف، بدون كسر أي وظيفة.

0. **[معلّق من إدخال سابق غير دقيق]** `src/components/Sidebar.jsx` — لم يُحوَّل بعد فعليًا، لا يزال 100% inline styles. يحتاج تحويل حقيقي (وليس توثيق فقط).
1. ~~`src/pages/SubjectList.jsx` — تحويل صندوق البحث، قائمة المواد، وزر المفضلة.~~ ✅ **منجز 2026-07-07** (انظر قسم 2 أعلاه).
2. `src/pages/Subject.jsx` — تحويل صفحة عرض المادة كاملة (العناوين، الأقسام، رسالة "لم تتم إضافة ملفات بعد"، قوائم المحاضرات).
3. `src/pages/StudyPlan.jsx` — تحويل الجدول (`CoursesTable`) وكتل المستويات القابلة للطي (`LevelBlock`).
4. `src/App.jsx` — تحويل الحاوية الرئيسية (`display: flex, direction: rtl`).
5. مراجعة نهائية: التأكد إن ما تبقى أي inline style غير ضروري، وحذف أي CSS متكرر أصبح بلا فائدة من `App.css` / `index.css`.
6. **معيار القبول لكل خطوة:** `npm run build` ينجح، والتصميم يبقى مطابق بصريًا (أو أفضل) للسابق، مع دعم `dir="rtl"` بشكل صحيح. **يُضاف الآن:** يجب التحقق الفعلي (بالبناء وبفحص ملف الـ CSS الناتج) قبل تسجيل أي بند كـ "✅ تم" في هذا الملف — لتفادي تكرار ما حصل مع `Sidebar.jsx` أعلاه.

### المرحلة ب: عارض PDF داخلي (مدمج بالتطبيق)
حاليًا: الملفات تُفتح بـ `<a href target="_blank">` فقط.

1. اختيار مكتبة عرض PDF مناسبة لـ React (مثل `react-pdf` أو تضمين `<iframe>`/`<embed>` كحل أبسط أول مرحلة).
2. بناء مكوّن `PdfViewer.jsx` يفتح كـ modal أو صفحة مدمجة بدل تبويب جديد.
3. ربطه من `Subject.jsx` بدل رابط `target="_blank"` الحالي.
4. التعامل مع حالة `file: ""` (لا يوجد ملف بعد) بنفس الرسالة الحالية "لا يوجد ملف بعد".
5. اختبار العرض على الموبايل (يرتبط بالمرحلة ج أدناه).

### المرحلة ج: التصميم المتجاوب (Responsive) للهاتف
1. تحويل `Sidebar` لقائمة قابلة للطي/hamburger menu على الشاشات الصغيرة.
2. جعل `App.jsx` layout يتحول من `flex row` إلى `flex column` تحت breakpoint معيّن (باستخدام Tailwind `md:` مثلًا).
3. اختبار جداول `StudyPlan.jsx` على شاشات ضيقة (قد تحتاج تمرير أفقي أو إعادة تصميم كبطاقات).

### المرحلة د: رفع محتوى PDF الفعلي
1. تنظيم مجلدات `public/pdf/<subjectId>/` بحسب كل مادة مفعّلة في `subjects-index.json`.
2. رفع الملفات الفعلية وتحديث حقول `file` في كل `lectures.json` من `""` إلى المسار الصحيح.
3. التأكد من حجم الملفات وتأثيره على حجم build/الاستضافة (قد يحتاج استضافة خارجية للملفات الكبيرة بدل تضمينها بالمستودع).

### المرحلة هـ (مستقبلية / اختيارية)
- إضافة مواد جديدة (subject.json + lectures.json لكل مادة غير "database" حاليًا).
