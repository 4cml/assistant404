## 1. نظرة عامة على المشروع

مشروع **Assistant404**: تطبيق ويب (React + Vite) لعرض مواد الخطة الدراسية ومحاضراتها (ملفات PDF)، مع دعم:
- تصفح المواد (الكل / المفضلة / الأخيرة / الأكثر زيارة)
- بحث فوري (Fuse.js)
- عرض الخطة الدراسية كاملة مع الربط بصفحات المواد
- صفحة مادة تعرض المحاضرات مقسّمة (نظري / عملي / ملفات إضافية / أسئلة)

**البنية التقنية:**
- `src/App.jsx` — الراوتنغ الرئيسي
- `src/components/Sidebar.jsx` — القائمة الجانبية
- `src/pages/SubjectList.jsx`, `Subject.jsx`, `StudyPlan.jsx`
- `public/data/study-plan.json` — الخطة الدراسية الكاملة
- `public/data/subjects-index.json` — فهرس المواد المفعّلة
- `public/pdf/<subjectId>/subject.json` + `lectures.json` — بيانات كل مادة ومحاضراتها

## 5. كيفية التعامل مع هذا الملف

- بعد إنجاز أي بند: يُنقل من قسم "قيد التنفيذ" إلى "المنجز" مع تاريخ اليوم وتفاصيل ما تم فعليًا (ليس مجرد "تم" بدون تفاصيل).
- إذا ظهرت مشكلة جديدة أثناء العمل: تُضاف كبند جديد تحت "المطلوب إنجازه" بنفس تنسيق الجدول/القوائم أعلاه.
- **قاعدة (أُضيفت 2026-07-07):** أي بند يُسجَّل كـ "✅ تم" يجب أن يكون قد تم التحقق منه فعليًا (بالبناء، أو بفحص الملف/الكود مباشرة) — وليس افتراضًا أو نية. إذا تبيّن لاحقًا أن بندًا مسجَّلًا كـ "تم" لم يكن صحيحًا، يُصحَّح فورًا مع ذكر أنه تصحيح، بدل حذف الأثر بصمت.
- **توضيح إضافي (أُضيف 2026-07-07، تحديث ثالث):** "التحقق الفعلي" يعني تحديدًا `npm run build` على الريبو الحقيقي + فحص CSS الناتج. تحقق syntax فقط (مثل Babel) أو مراجعة كود بصرية **لا يكفي وحده** لتسجيل بند كـ "✅ تم" نهائي — يُسجَّل كبند "قيد التنفيذ" بحالة "الكود مُعدَّل، بانتظار تحقق فعلي" لحين توفر الريبو الكامل والتحقق منه.
