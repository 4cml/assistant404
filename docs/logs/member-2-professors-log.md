## [التاريخ: 2026-07-08]

### ما تم إنجازه
- إنشاء `src/lib/professorVariants.js` بأربع دوال: `hasProfessorVariants`, `getActiveVariant`, `getAllVariants`, `resolveLecturesFile`.
- تعديل `src/pages/Subject.jsx`:
  - استيراد `getActiveVariant` و `resolveLecturesFile`.
  - إعادة هيكلة تسلسل التحميل بحيث يتم تحديد ملف المحاضرات (`lectures.json` الافتراضي أو `lectures-{professorId}.json`) *بعد* معرفة محتوى `subject.json`، بدل ما كان يُجلب `lectures.json` بشكل مستقل ومتوازي.
  - إضافة عرض اسم الدكتور الفعّال (`الدكتور: ...`) تحت معلومات المادة، يظهر فقط إذا المادة فيها `professorVariants`.
- تحقق فعلي (build + اختبار node مباشر لملف `professorVariants.js`) على 3 حالات: مادة فيها `professorVariants` نشطة، مادة قديمة بدون `subject.json` إطلاقاً، ومادة فيها `subject.json` لكن بدون `professorVariants`. الثلاثة رجّعوا السلوك الصحيح.
- ملاحظة: لا يوجد حالياً ولا `subject.json` واحد فعلي بالمشروع (كل المواد لسا تعتمد على `study-plan.json` + `lectures.json` فقط) — التوافق العكسي أساسي جداً هون وليس حالة نادرة.

### مشكلات واجهتها
- لا يوجد.

### قرارات اتخذتها (ولماذا)
- لو مادة فيها `professorVariants` لكن ولا نسخة `active:true` (خطأ بيانات محتمل من لوحة تحكم العضو 1)، `getActiveVariant` ترجع أول عنصر بدل صفحة فاضية للطالب — قرار دفاعي لتجنب كسر التجربة.
- منطق اختيار ملف المحاضرات معزول بالكامل بـ `professorVariants.js` (مو داخل `Subject.jsx`) عشان يسهل اختباره لوحده ويقلل التغيير بالملف المشترك لأقل قدر ممكن.

### هل احتجت تعديل ملف مشترك؟ (نعم/لا) — إذا نعم، هل تم التنسيق مع الإدارة أولاً؟
- نعم، `src/pages/Subject.jsx`. **لم يتم التنسيق المسبق مع الإدارة قبل التنفيذ** — التعديل جاهز كمسودة للمراجعة وليس مدفوعاً (push) للريبو. يُرجى تنسيق هذا مع العضو 3 (الوضع المظلم) قبل الدمج لأنه سيلمس نفس الملف.

### ملاحظات للأعضاء الآخرين
- للعضو 1: شكل `professorVariants` بـ `subject.json` جاهز بالضبط متل ما موثّق بخطة الفريق. لوحة التحكم تحتاج فقط تكتب/تعدّل هذا الحقل + تولّد ملفات `lectures-{professorId}.json` منفصلة — منطق القراءة جاهز من طرفي.
- للعضو 3: تعديلي على `Subject.jsx` كان محصور بمنطق التحميل (useEffect) وسطر عرض واحد (`<p>الدكتور: ...</p>`) — ما لمست أي inline style موجود، فالتعارض المتوقع بينا محدود لكن لازم مراجعة سريعة قبل الدمج.
- للعضو 4: لسا ولا مادة فيها `subject.json` فعلي بالمشروع — ممكن يفيدك هاد بتدقيق `id-mapping-audit.md`.
